# DEMO_SCRIPT.md — سناریوی دموی گام‌به‌گام

مدت تقریبی: ۱۵ دقیقه. قبل از شروع: `docker compose up -d && pnpm setup && pnpm build && pnpm start` و باز کردن `http://localhost:3000`.

## حساب‌های دمو (رمز همه: `Admin@12345678`)

| نقش | ایمیل |
|-----|-------|
| ادمین سیستم | admin@example.edu |
| استاد | rezai@example.edu |
| Head TA | headta@example.edu |
| دانشجو | student@example.edu |

## پرده ۱ — استاد: ساخت فرصت TA با فرم‌ساز (۳ دقیقه)

1. با حساب استاد وارد شوید → داشبورد استاد (KPIها، درخواست‌های اخیر، هشدار نبود Head TA اگر باشد).
2. «فرصت‌های TA» → «ایجاد فرصت جدید».
3. عنوان/توضیح/شرایط را پر کنید؛ در «تنظیم فرم درخواست»: شماره دانشجویی + معدل + نمره‌ی قبلی را تیک بزنید، یک فیلد سفارشی «لینک گیت‌هاب» (الزامی) اضافه کنید. نکته‌ی گفتنی: فیلدهای آماده از داده‌ی واقعی سامانه خوانده می‌شوند و دانشجو نمی‌تواند آن‌ها را جعل کند.
4. مهلت آینده بدهید (نمایش زنده‌ی تاریخ شمسی زیر فیلد) و ثبت کنید.

## پرده ۲ — دانشجو: درخواست + تحویل تکلیف (۳ دقیقه)

1. خروج، ورود با حساب دانشجو.
2. همان فرصت را باز کنید؛ اول بدون پر کردن «لینک گیت‌هاب» ثبت بزنید → خطای فیلد الزامی (متن تایپ‌شده پاک نمی‌شود). فیلد را پر کنید → ثبت موفق.
3. «نمرات» → بخش «تکالیف با مهلت تحویل» → روی تکلیف موعددار «تحویل فایل» بزنید و فایلی بدهید. اگر مهلت گذشته باشد برچسب «دیرکرد» می‌بینید.

## پرده ۳ — استاد: بررسی متقاضی، پذیرش، خروجی اکسل (۳ دقیقه)

1. ورود مجدد استاد → فرصت → «بررسی متقاضیان».
2. روی «اطلاعات فرم» بزنید: شماره دانشجویی/معدل واقعی + پاسخ فیلد سفارشی.
3. «خروجی اکسل» را دانلود کنید (ستون‌ها = فیلدهای فعال فرم).
4. «قبول» بزنید → نقش TA به‌صورت تراکنشی ساخته می‌شود و نوتیفیکیشن پیشنهاد ساخت گروه می‌آید.

## پرده ۴ — گروه‌های ارتباطی و پنل درس (۲ دقیقه)

1. وارد پنل درس شوید (از داشبورد یا لینک نوتیفیکیشن).
2. در کارت «گروه‌های ارتباطی»: «ساخت/همگام‌سازی گروه تیم TA» → پیام موفقیت با تعداد اعضا. دوباره بزنید → «گروه از قبل کامل است» (idempotent).
3. کارت «فایل‌های آموزشی»: یک جزوه آپلود کنید؛ بگویید TA فقط با مجوز اعطایی استاد می‌تواند آپلود کند.

## پرده ۵ — دفتر نمرات: import، قفل، تجدیدنظر (۲ دقیقه)

1. «دفتر نمرات» درس → ورود گروهی نمره از اکسل با پیش‌نمایش خطاها → ثبت.
2. روی آیتم نمره «🔓 باز» بزنید تا قفل شود → توضیح دهید از این لحظه هیچ‌کس (حتی import) نمی‌تواند نمره را عوض کند تا استاد قفل را باز کند.
3. انتشار نمره → با حساب دانشجو نمره + دکمه‌ی «تجدیدنظر» را نشان دهید.

## پرده ۶ — صف رفع اشکال + سلامت سیستم (۲ دقیقه)

1. دانشجو: «رفع اشکال» → جلسه → «صف رفع اشکال» → «پیوستن به صف» (جایگاه در صف دیده می‌شود).
2. Head TA: همان صفحه → «صدا زدن نفر بعد» → سمت دانشجو (با polling چند ثانیه‌ای) «نوبت شماست!» ظاهر می‌شود.
3. ادمین: پنل ادمین → کارت «سلامت سیستم» (وضعیت سبز DB/Redis/Storage + تاخیر).

## نکات پایانی برای پرسش‌وپاسخ

- دارک‌مود و نمای موبایل را یک‌بار toggle کنید (ماندگار است).
- اگر درباره‌ی امنیت پرسیدند: `SECURITY_NOTES.md` خلاصه است، `docs/security-checklist.md` مورد به مورد.
- اگر صفحه‌ای در حالت dev کند بود، توضیح: کامپایل on-demand مخصوص dev است؛ دمو را روی `pnpm start` اجرا کنید.
