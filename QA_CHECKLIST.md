## لایه ۳ — ممیزی زنده‌ی دستی (مرورگر واقعی + بررسی مستقیم دیتابیس)

برای هر قابلیت جدید، سناریوی کامل با حساب‌های seed اجرا و نتیجه با query مستقیم Postgres تایید شده است:

| قابلیت | سناریوی اجراشده | تایید DB |
|--------|-----------------|----------|
| تحویل وظیفه‌ی TA | TA فایل داد → Head TA در Kanban دانلود کرد | ردیف `TaskSubmission` |
| تحویل تکلیف دانشجو | دانشجو فایل داد؛ تحویل بعد از مهلت → برچسب «دیرکرد» | ردیف `AssignmentSubmission` + `late: true` |
| گروه‌های ارتباطی | Head TA گروه تیم و گروه کل درس ساخت؛ کلیک دوم = sync بدون duplicate؛ دانشجو 403 گرفت | ردیف‌های `MessageThread`/`Participant` |
| فرم‌ساز درخواست TA | استاد فرم با فیلد سفارشی الزامی ساخت؛ دانشجو بدون آن رد شد، با آن ثبت شد؛ رزومه‌ی غیرفعال حتی با ارسال دستی ذخیره نشد؛ خروجی اکسل دانلود شد | `formConfigJson`/`customFieldsJson` |
| کتابخانه فایل درس | استاد آپلود؛ دانشجو فقط دانلود؛ TA بدون مجوز 403؛ با مجوز `permissionsJson` موفق؛ لغو مجوز فوراً اثر کرد | ردیف `CourseMaterial` |
| صف آفیس‌آور | دانشجو پیوست → میزبان صدا زد → پایان؛ صدازدن توسط دانشجو 403 | وضعیت `OfficeHourQueueEntry` |
| یادآوری مهلت | باز کردن داشبورد نوتیفیکیشن DEADLINE ساخت؛ بار دوم duplicate نساخت (idempotent) | شمارش `Notification` |
| باطل شدن session بعد از تغییر رمز | تغییر `passwordChangedAt` → session قبلی فوراً null شد؛ login مجدد سالم | — |
| قفل نمره | آیتم قفل → ثبت نمره 409؛ باز → 201 | مقدار `lockedAt` |
| اعتبارسنجی فرصت | `opensAt` بعد از `deadline` → 400؛ مهلت گذشته → 400 | — |
| CHECK ترم | INSERT ترم با پایانِ قبل از شروع → خطای constraint | مستقیم psql |
| Migration از صفر | زنجیره‌ی کامل migrationها روی دیتابیس خام | `_prisma_migrations` |
| Idempotency seed | دوبار `pnpm db:seed` بدون خطا/تکرار | شمارش ردیف‌ها |
